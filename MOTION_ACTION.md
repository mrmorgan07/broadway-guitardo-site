Action/motion-эффекты для red_draft3 — ранжированная подборка
Логика приоритета: эффект ценнее, если он (а) попадает в готику-кровь, (б) дёшев на мобайле — анимирует только transform/opacity/background-position, не filter/box-shadow в цикле, (в) уже опирается на существующую инфраструктуру (reveal-on-scroll observer, hover-подъём).

1. Specular-sweep по заголовкам секций и слогану (перенос фирменного приёма с логотипа)

Где: .section-title, .afisha-title, .slg__row span (крупный красный текст слогана).
Техника: тот же background-clip: text + бегущая узкая полоса через background-position, что и в hero/логотипе — но не зацикленный, а одноразовый, по входу в вьюпорт. Вешаем класс через существующий IntersectionObserver (тот же, что reveal): при .in-view запускаем animation: titleSweep 1.1s ease-out (один проход, forwards). Для слогана — --red фон градиента, ядро --red-bright+#fff4ec.
Результат: заголовок «проявляется», и по нему один раз проходит кроваво-холодный отблеск, как клинок по тексту. Бренд читается как единая система (логотип → заголовки → слоган).
Перф: копеечный. Один background-position, один проход, без повтора. Мобайл — идеально.
Вкусно vs риск: очень вкусно, риск минимальный — потому что разово, а не мигает постоянно. Это ТОП-кандидат.
2. «Кровь в тенях» — направленный red bloom при hover карточек афиши/спектакля

Где: .poster, .leader, .feature__banner.
Техника: у вас уже есть translateY(-8px) + box-shadow. Добавляем к hover-shadow бордовый слой и тонкую красную каёмку через псевдоэлемент: .poster::after { box-shadow: inset 0 0 0 1px transparent; } → на hover border-color/inset уходит в rgba(158,36,54,.5), плюс к внешней тени добавляем 0 24px 60px rgba(120,6,6,.28). Переход box-shadow .4s (уже есть). На .poster__date (красный бейдж) — лёгкий box-shadow: 0 0 18px rgba(232,18,31,.4) на hover карточки.
Результат: карточка не просто всплывает, а будто наливается кровью из темноты по краю. Тематично, дорого.
Перф: box-shadow-переход только на hover (не цикл) — на desktop ок. На мобайле hover не триггерится — деградация бесплатная, что хорошо.
Вкусно vs риск: вкусно, риск низкий, если держать alpha теней ≤0.3. Передержать = яркий красный glow по всему периметру (казино).
3. Цвет, проявляющийся на ч/б фото «Лиц коллектива»

Где: .leader__photo img (сейчас filter: grayscale(1), переход уже заложен на 0.5s).
Техника: на hover (desktop) — filter: grayscale(0.15) contrast(1.08) + очень слабый бордовый дуотон через наложенный .leader__photo::after { background: var(--red); mix-blend-mode: color; opacity: 0; } → opacity: .12 на hover. Лицо «оживает» из мёртвого ч/б в чуть кровящий цвет.
Результат: портрет будто пробуждается — идеальная вампирская метафора, и при этом строго.
Перф: filter на одном изображении по hover — приемлемо на desktop; mix-blend-mode статичен (анимируем только opacity). Мобайл: hover нет → остаётся ч/б, ноль стоимости.
Вкусно vs риск: очень вкусно концептуально. Риск: grayscale-переход на больших фото на слабых машинах может дёргаться — поэтому только hover (разово), не scroll-driven.
4. Reveal с «подъёмом из темноты» (clip-reveal) для заголовков и медиа

Где: усиление существующего reveal-on-scroll для .feature__banner, .poster__media, .afisha-title.
Техника: к текущему blur-in добавить clip-path: inset(0 0 100% 0) → inset(0 0 0 0) (раскрытие снизу вверх, как поднимающийся занавес — рифмуется с loader-занавесом) и/или mask со сдвигом. Запускается тем же observer'ом. Для текста — построчный stagger через transition-delay на nth-child.
Результат: контент не «появляется», а выезжает из тьмы, поддерживая занавесную метафору всего сайта.
Перф: clip-path/opacity — дёшево, GPU-friendly, разово. Мобайл ок.
Вкусно vs риск: вкусно, риск низкий. Передержать = слишком долгие задержки stagger (>120ms на элемент) → сайт «тормозит». Держать каскад быстрым.
5. Тонкая виньетка-переход между секциями (light leak / breathing seam)

Где: границы секций hero→afisha→feature→slogan.
Техника: на каждую секцию position: relative + псевдоэлемент сверху/снизу: linear-gradient(180deg, transparent, rgba(10,8,9,.9)) или узкая бордовая «жила» radial-gradient по центру шва, opacity слегка дышит через очень медленный @keyframes (8–10s, prefers-reduced-motion гасит). Альтернатива без анимации — просто статичные затемнённые швы, чтобы секции «тонули» друг в друге.
Результат: нет резких стыков; сайт читается как непрерывная тёмная сцена, кое-где тлеющая бордо.
Перф: статичный вариант — ноль. Дышащий opacity — дёшево, но это цикл, поэтому на мобайле/saveData отключать.
Вкусно vs риск: вкусно как «клей» композиции. Риск передержать невелик; главное — не делать видимых пульсирующих полос.
6. Магнитные стрелки каруселей + «накал» CTA

Где: .carousel__arrow, .feature__arrow, .pcar__arrow; CTA-кнопки --cta.
Техника: у вас уже есть магнитные кнопки — распространить тот же JS на стрелки. Для главной CTA — на hover не просто смена фона, а короткий «накал»: background-position бегущего блика по кнопке + box-shadow glow rgba(232,18,31,.4) (уже частично есть в feature). Активная точка карусели .pcar__dot.active — добавить слабый pulse-ring (один box-shadow scale, медленно).
Результат: управление ощущается «живым», CTA притягивает не мигая.
Перф: магнит — это transform по mousemove (throttle через rAF), desktop-only; на мобайле не вешаем. Дёшево.
Вкусно vs риск: вкусно при сдержанной амплитуде магнита (≤8px). Риск: сильный магнит + glow = «игрушечность».
7. Parallax-дрейф фоновых медиа (очень слабый)

Где: .feature__banner img, .poster__media img, hero-фон.
Техника: scroll-linked transform: translateY() через rAF + единый scroll-листенер (или CSS animation-timeline: scroll() там, где поддержано, с фолбэком). Амплитуда 4–8%.
Результат: глубина сцены, кинематографичность.
Перф: scroll-driven transform — самый дорогой в списке на мобайле (постоянный пересчёт при скролле). Только transform, обязательно rAF-throttle, отключать на мобайле/saveData/слабом CPU.
Вкусно vs риск: вкусно, но риск перфа высок на мобиле, а у вас мобайл в приоритете — поэтому в самом низу.
ТОП-3 «максимум вау / минимум китча» — внедрять первыми
Specular-sweep по заголовкам секций и слогану (№1). Разовый, по входу во вьюпорт. Почти бесплатно, мгновенно делает весь лендинг единой системой с логотипом и hero, без единого зацикленного мигания. Лучшее соотношение вау/стоимость/риск.

«Кровь в тенях» при hover карточек (№2). Точное попадание в тему (наливается кровью из темноты), переиспользует уже существующие hover-переходы, на мобайле деградирует в ноль автоматически. Низкий риск, высокий эффект на desktop.

Цвет, проявляющийся на ч/б портретах «Лиц коллектива» (№3). Самая сильная концептуальная метафора («оживание/пробуждение»), при этом строго и тихо. Стоит делать третьим, потому что нужен аккуратный тюнинг grayscale-перехода, но визуально это запоминается сильнее всего.

Общий принцип для всех: разово по триггеру (scroll-in / hover), а не в бесконечном цикле; в цикле на мобайле оставляем только то, что уже есть (угли/зерно). Все три топ-эффекта обязательно под prefers-reduced-motion: reduce отдаём статичную «дорогую» версию (заголовок с фиксированным бликом по центру, карточка с готовой бордовой каёмкой, фото — лёгкий не-ч/б стейт), и переиспользуем ваш существующий флаг слабого устройства, чтобы глушить sweep/blend на бюджетных Android.

Возвращаясь к исходной задаче (сверкание заголовка hero) — она закрыта выше: рекомендация была комбо «кровавый specular-sweep + статичная гравировка» на .hero__title/.hero__brandsub, с готовым CSS-наброском, обработкой prefers-reduced-motion, мобильной деградацией и согласованием со спотлайтом/углями/виньеткой. Подборка для остальных компонентов выстроена как прямое расширение этого же фирменного приёма.
