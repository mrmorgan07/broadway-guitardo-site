Технологический стек
Компонент	Технология	Причина выбора
Framework	Next.js 14 (App Router)	SSR для SEO, быстрая загрузка, отличный DevEx
Язык	TypeScript (strict mode)	Типобезопасность, самодокументируемость
UI	React 18	Серверные и клиентские компоненты
Стили	Tailwind CSS	Быстрая вёрстка, единый дизайн-система
Компоненты	shadcn/ui (Radix UI)	Кастомизируемые, доступные, красивые
Анимации	Framer Motion	Плавные, премиальные анимации
База данных	SQLite + Drizzle ORM	Лёгкая, файловая, идеально для MVP
Авторизация	NextAuth.js (Credentials)	Простая и безопасная
Валидация	Zod	TypeScript-first валидация
Формы	React Hook Form	Производительность, интеграция с Zod
Загрузка файлов	React Dropzone + Sharp	Drag-and-drop, оптимизация изображений

Архитектурные принципы
Feature-based структура: Группируем по функциональности, а не по типам файлов.

Server Components по умолчанию: Используем серверные компоненты везде, где нет интерактивности.

API-first подход: Все данные получаем через API Routes, фронтенд — только клиент.

Мобильный-first: Адаптивность с самого начала.

Типобезопасность: Zod для валидации форм и API, TypeScript для всего остального.
