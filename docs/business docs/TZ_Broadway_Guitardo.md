## 1. Общие сведения

### 1.1 Назначение системы

Публичный одностраничный сайт-витрина (лендинг) театрально-хорового коллектива, наполняемый через собственную административную панель (CMS). Сайт демонстрирует репертуар мюзиклов, ведёт к покупке билетов и принимает интерес к участию в хоре.

### 1.2 Бизнес-цели

1. **Продажа билетов** — привести зрителя к покупке (текущая реализация: переход по внешней ссылке из карточки спектакля).
2. **Рекрутинг в хор** — привлечь новых участников коллектива (секция «Присоединиться» + кнопка «В хор» в мобильном баре).
3. **Премиальный имидж** — восприятие коллектива как профессионального и атмосферного через готическую визуальную подачу и моушн-эффекты.

### 1.3 Целевая аудитория

- **Зрители** — потенциальные покупатели билетов на мюзиклы (Москва и окрестности).
- **Кандидаты в хор** — желающие петь и участвовать в постановках.
- **Контент-менеджер** — администратор, наполняющий сайт через CMS без участия разработчика.

### 1.4 Термины и глоссарий

| Термин | Значение |
|---|---|
| **Лендинг** | Публичная одностраничная витрина (актуально — `red_draft3_1_1`). |
| **CMS / Админка** | Панель управления контентом на `/admin`. |
| **Hero** | Первый («главный») экран лендинга с названием, фоном/видео и обратным отсчётом. |
| **Featured / текущий проект** | Спектакль, чьё название совпадает с заголовком Hero; иначе — первый в списке. |
| **Афиша** | Карусель карточек всех спектаклей. |
| **Солист** | Объект `{name, role, photo}` внутри спектакля. |
| **Медиатека** | Раздел CMS для управления всеми загруженными фото и видео. |
| **db.json** | Файл-хранилище всего контента сайта. |
| **CSRF-токен** | Токен защиты мутирующих запросов админки. |
| **reduced-motion** | Системная настройка «уменьшить движение» (`prefers-reduced-motion`). |

## 2. Архитектура и стек

### 2.1 Компоненты системы

| Компонент | Расположение | Описание |
|---|---|---|
| **Публичный фронт** | `/red_draft3_1_1/` (`index.html`, `script.js`, `style.css`) | Статический HTML + vanilla JS, рендерит контент из `/api/content`. |
| **Админ-CMS** | `/frontend/admin/` (`admin.html`, `js/admin.js`, `css/admin.css`) | Vanilla JS + Bootstrap 5.3, редактирует все секции и медиа. |
| **Бэкенд-API** | `/backend/server.js` | Express REST API: контент, авторизация, загрузка/оптимизация медиа. |
| **Хранилище контента** | `/backend/data/db.json` | JSON-файл со всем контентом (атомарная запись). |
| **Хранилище доступа** | `/backend/data/auth.json` | Логин + bcrypt-хэш пароля. |
| **Файловое хранилище** | `/backend/uploads/` (+ `thumbs/`, `videos/`) | Загруженные фото, превью, видео. |

> **Исторический контекст.** В репозитории есть параллельные макеты-итерации (`v2`–`v5`, `red_draft`, `red_draft2`, `red_draft3`, `red_draft3_1`, `red_draft3_1_1`), все отдаются статикой с одного бэкенда через общий `/api/content`. Актуальная публичная версия — **`red_draft3_1_1`**.

### 2.2 Технологии

| Слой | Технология |
|---|---|
| Бэкенд-сервер | Node.js + **Express** |
| Загрузка файлов | **Multer** (disk storage) |
| Обработка изображений | **Sharp** (даунскейл ≤2560px, перекод, EXIF-поворот, превью 400×400) |
| Авторизация | **bcryptjs** + самописные сессии в памяти + CSRF-токены |
| Хранилище | JSON-файл (`db.json`) с атомарной записью (tmp + rename) |
| Публичный фронт | Чистый HTML + CSS + **vanilla JavaScript** (без сборки) |
| Шрифты | Druzhok (заголовки), Avenir Next (текст), Futura (лейблы); фолбэк — Playfair Display |
| Админ-фронт | **Bootstrap 5.3** + vanilla JavaScript |
| Карты | Яндекс.Карты (iframe map-widget) |

> **Расхождение со стеком из CLAUDE.md.** Декларированный там стек (Next.js 14, TypeScript, Drizzle, NextAuth) **не используется**: реальная реализация — Express + JSON + vanilla JS без сборки. Это сознательное решение MVP-этапа (см. раздел 9).

### 2.3 Схема взаимодействия (текстом)

```
[ Браузер зрителя ]
      │  GET /red_draft3_1_1/        (статика: html/css/js)
      │  GET /api/content            (весь контент одним JSON)
      │  GET /api/gallery/photos     (список фото для галереи)
      ▼
[ Express (backend/server.js) ] ── читает ──▶ [ db.json ]
      ▲                          ── отдаёт ──▶ [ /uploads/* статикой ]
      │
[ Браузер админа ] ── /admin (Bootstrap SPA)
      │  POST /api/login            → token + csrfToken (в sessionStorage)
      │  PUT/POST/DELETE /api/*      (X-Auth-Token + X-CSRF-Token)
      │  POST /api/upload(/video)    (Sharp-оптимизация → uploads/)
      ▼
[ db.json + uploads/ ]  (атомарная запись JSON, проверка «файл используется»)
```

Поток данных: контент-менеджер правит данные в админке → бэкенд пишет в `db.json` → зритель при загрузке лендинга получает свежий `/api/content` → JS рендерит секции на клиенте.

### 2.4 Окружение и деплой

- **Запуск:** `npm start` из корня (бэкенд слушает `PORT`, по умолчанию `3000`).
- **Окружение:** переменные из корневого `.env`.
- **Деплой:** Render free-tier. ⚠️ **Диск эфемерный** — загруженные через CMS медиа и правки `db.json` живут только до следующего деплоя (в git вернётся лишь закоммиченное).
- **Cold start:** первый запрос после простоя ждёт ~30–50 с пробуждения инстанса.

### 3.3 Бэкенд / API (`backend/server.js`)

Мутирующие запросы требуют `requireAuth` (`X-Auth-Token` или cookie `authToken`) и `requireCsrf` (`X-CSRF-Token`).

#### Публичные эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| GET | `/api/content` | Весь контент сайта (агрегат `db.json`). |
| GET | `/api/projects` | Список спектаклей. |
| GET | `/api/projects/:id` | Один спектакль по ID. |

### 7.1 Публичный фронт (`red_draft3_1_1`)

- Полностью клиентский рендер всех секций из `/api/content`: Hero, О коллективе, Слоган, Лица коллектива, Афиша, Текущий проект, Галерея, Присоединиться, Локация, Футер.
- Hero с фоновым видео/изображением, кнопками «Смотреть»/«Трейлер».
- **Обратный отсчёт** до премьеры с парсером русских дат (родительный падеж месяцев).
- Карусели: фото «о коллективе», афиша (десктоп — окно из 3, мобайл — scroll-snap), баннер featured, галерея спектакля.
- Модалки спектакля и трейлера (локальное видео / YouTube), закрытие по Escape/оверлею/крестику.
- Слоган с асимметричным выравниванием и красными линиями.
- Встроенная Яндекс.Карта с кастомной красной меткой.
- Футер с контактами и соцсетями (SVG-иконки), ссылка на админку.

### 7.2 Эффекты и анимации

- **Занавес-загрузка** (red_draft2): створки, лампочки-marquee, прогресс, удержание 1500/700 мс, пропуск по клику.
- **Спотлайт-курсор** и **тлеющие угли** (canvas) в Hero (red_draft3) с деградацией под reduced-motion / Save-Data / слабые устройства.
- **Мобильный липкий бар** «Билеты»/«В хор» с появлением после Hero и скрытием у футера.
- Эффект **#1 specular-sweep** заголовков/слогана.
- Эффект **#3 «кровь в тенях»** — кровавый дуотон и оживание ч/б портретов при hover.
- Эффект **#4 clip-reveal** медиа афиши и featured «из тьмы».
- **Магнитные кнопки** на десктопе.
- Reveal-анимации появления секций при скролле.
- Цветовая палитра «холодный малиновый» `#db4a62` с упором на читаемость на мобиле (red_draft3_1).

### 7.3 Админка (CMS)

- Авторизация (логин/пароль), хранение токена/CSRF в `sessionStorage`, проверка сессии, выход.
- Баннер о пароле по умолчанию + раздел смены пароля.
- Защита несохранённых изменений (`isDirty` + `beforeunload`).
- Редактирование всех секций контента (Hero, О нас, Худрук, Концертмейстер, В хор, Контакты, Локация, Галерея).
- CRUD спектаклей с редактором (поля, drag-and-drop сортировка галереи, солисты с фото).
- Медиатека: просмотр/загрузка/удаление фото и видео, прогресс загрузки видео (XHR), копирование URL, защита от удаления используемого файла.
- Живой предпросмотр карты в разделе «Как добраться».
- Мобильная навигация (off-canvas сайдбар), toast-уведомления.

### 7.4 Бэкенд

- Express REST API: `/api/content`, секционные GET/PUT, CRUD `/api/projects`, авторизация, загрузка/медиатека.
- Авторизация: bcrypt + самописные сессии (TTL 24 ч) + CSRF; флаг `mustChangePassword`.
- Загрузка изображений с Sharp-оптимизацией (даунскейл 2560px, перекод, EXIF-поворот, превью).
- Загрузка видео до 500 МБ.
- Медиатека с проверкой «файл используется» и защитой path-traversal.
- Атомарная запись `db.json` и `auth.json`.

### 7.5 Инфраструктура

- Единый Express-сервер раздаёт все версии макетов (`v2`–`v5`, `red_draft*`) и админку.
- Тематическая 404-страница.
- Конфигурация через `.env` (`PORT`).
- Деплой на Render free-tier (с известным ограничением эфемерности).

## 9. Ограничения и риски

| Риск / ограничение | Описание | Митигация |
|---|---|---|
| **JSON вместо БД** | Параллельные записи дают гонки; пользовательские заявки нельзя хранить в JSON. | Атомарная запись есть; заявки — только в БД (M1). |
| **Render free-tier — эфемерный диск** | Загруженные медиа и правки CMS пропадают при деплое. | R2 для медиа + Postgres для данных (M1). |
| **Cold start** | Первый запрос после простоя ждёт 30–50 с. | Платный план или внешний «пингер». |
| **Vanilla без сборки** | Нет TypeScript/типобезопасности, нет бандлинга; расхождение со стеком из `CLAUDE.md`. | Осознанный выбор MVP; миграция на Next.js — отдельная инициатива. |
| **Сессии в памяти** | Сбрасываются при рестарте/сне инстанса — админ разлогинивается. | Персистентные сессии (M6). |
| **152-ФЗ при формах** | Форма с email/телефоном требует политики конфиденциальности и согласия на обработку ПДн. | Cookie-баннер + политика до запуска формы заявки. |
| **Клиентский рендер и SEO** | Поисковики/мессенджеры не видят серверных мета. | Серверный рендер мета (M5). |
